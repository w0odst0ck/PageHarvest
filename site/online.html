🌐 在线采集部署指南

面向开发者 — 在线采集代码全部公开，用户自行部署

📋 策略说明

PageHarvest 的 离线分析 功能是面向终端用户的"产品"体验， 而 在线采集 部分的代码全部公开在 GitHub 仓库中， 供有技术能力的开发者自行部署和使用。

简单说：不会代码的用户用离线 exe / Pages 上传 ZIP 解析； 会代码的开发者可以自建在线采集环境，获取实时数据。

📦 前置条件

  • Python 3.11+
  • Chrome 或 Chromium 浏览器（用于 Selenium 或 Playwright 驱动）
  • 基本的 Python 环境和命令行操作能力
  • 对目标电商平台的反爬策略有一定了解

🔧 安装步骤

  1. 克隆仓库： 
    git clone https://github.com/w0odst0ck/PageHarvest.git
cd PageHarvest
  2. 安装依赖： 
    pip install -r requirements.txt
  3. （可选）安装浏览器驱动：
    采集依赖 Selenium/Playwright，需要对应浏览器的 WebDriver 
    # 如果使用 Playwright
playwright install chromium

🚀 使用示例

在线采集搜索页

python -m cli.online --keyword 螺丝刀 --pages 3

采集指定 URL

python -m cli.online --url https://www.1688.com/... --mode detail

输出到指定目录

python -m cli.online --keyword 收纳盒 --pages 5 --out ./my_data/

离线解析已有 ZIP

python -m cli.offline input.zip --out ./results/

⚠️ 注意事项

  • 反爬机制：各平台均有不同程度的反爬限制，建议合理设置请求间隔
  • IP 问题：大规模采集建议使用代理 IP 轮换，避免 IP 被封
  • Cookie 配置：部分平台需要登录态，采集前需配置有效的 Cookie
  • 频率控制：请控制请求频率，避免对目标平台造成负担
  • 法律合规：请遵守目标平台的服务条款，仅用于个人学习和研究
  • 数据量：在线采集生成的 HTML 文件可用 CLI 离线解析，或上传到离线 exe 处理
