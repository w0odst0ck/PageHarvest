📖 使用指南

分步教程：从采集 HTML 到生成结构化报告

1 安装油猴脚本

在浏览器中安装 Tampermonkey 或 Violentmonkey 扩展，然后安装下方脚本。

脚本 平台 用途 安装
1688 自动翻页+保存 v1.4 🏭 1688 搜索页自动翻页、保存 HTML Greasy Fork 安装 →
震坤行 HTML 下载器 v9.2 🔩 震坤行 SPA 适配，自动翻页+保存 HTML Greasy Fork 安装 →
💡 1688 用户推荐：还可安装 1688 采购助手 浏览器插件，搜索页自动提取数据并导出 XLSX，上传 PageHarvest 数据比 HTML 更完整。
⚠️ 京东用户：由于反爬机制极严，无法通过油猴自动采集。请使用浏览器手动搜索 → 逐页 Ctrl+S 保存 → 打包上传。详情页解析仍支持。

2 在电商平台搜索产品

在目标平台搜索你想要分析的关键词。脚本会自动翻页采集所有搜索结果。

如果你的平台没有油猴脚本支持（如京东），手动操作即可：

  • 搜索目标关键词
  • 逐页浏览搜索结果
  • 用浏览器 Ctrl+S 保存完整页面
搜索页 HTML 保存示意图 — 对应仓库 web/static/img/1.jpeg

3 保存 HTML 文件

有两种获取 HTML 的方式：

方式 A：油猴脚本自动保存

  • 安装对应平台的油猴脚本后，脚本会自动翻页并保存页面 HTML
  • 文件默认保存在浏览器的下载目录中

方式 B：手动保存（适用于所有平台）

  • 打开商品详情页
  • 推荐使用 SingleFile 扩展保存完整页面（可保留图片 CDN 原始链接）
  • 或直接 Ctrl+S（Windows/Linux）/ Cmd+S（Mac）保存为 HTML
  • 详情页模式：多个页面打包在一起
详情页保存示意图 — 对应仓库 web/static/img/3.jpeg
💡 1688 详情解析：集成自开源库 jiyun/1688，支持 data-sf-original-src 提取图片 CDN 直链。

4 打包成 ZIP

将所有 HTML 文件放入一个文件夹，打包成 ZIP 压缩包。

  • 文件命名没有限制，程序会自动识别 HTML 文件
  • 一个 ZIP 中可以包含多个搜索页或详情页
  • 建议每个分析任务单独打包
  • ZIP 体积建议不超过 50MB

5 上传解析

两种方式任选其一：

方式 A：下载 exe，本地运行（推荐）

  1. 从 下载页面 获取 PageHarvest.exe
  2. 双击运行，浏览器自动打开 http://localhost:8080
  3. 在页面上选择模式：搜索页选品 或 详情页解析
  4. 拖拽或选择 ZIP 文件上传
  5. 等待解析完成（进度条自动刷新）

方式 B：（建设中）通过 GitHub Actions 在线解析

  1. 访问 PageHarvest GitHub 仓库
  2. 通过 workflow_dispatch 手动触发离线解析工作流
  3. 上传 ZIP 或提供文件直链
  4. 从 Actions Artifact 下载结果
解析上传界面示意图 — 对应仓库 web/static/img/2.jpeg
💡 CLI 模式：开发者也可直接通过命令行运行：
python -m cli.offline input.zip --out ./results/

6 下载报告

解析完成后，你可以下载以下格式的数据：

格式 内容 用途
CSV 结构化数据表格 Excel / WPS 直接打开，做数据分析
Excel 完整解析数据（详情页模式下所有字段展开） 需要查看完整属性、SKU 变体时使用
TXT 可读报告摘要 快速浏览分析结论
JSON 含原始解析数据的完整导出 二次开发或数据整合
💡 搜索页选品：输出推荐清单（按策略分层），含排名、价格、销量等关键字段。
详情页解析：输出品牌、价格、SKU 变体、属性、主图等结构化数据。
